⚕️ Avertissement — whispsy est un outil d'aide à la rédaction, pas un dispositif médical certifié. Les transcriptions et comptes-rendus générés doivent toujours être relus et validés par le praticien, qui reste seul responsable du contenu. Fourni « tel quel », sans aucune garantie.

①Qu'est-ce que c'est ?

whispsy transcrit des comptes-rendus médicaux dictés (au micro ou depuis un fichier audio) et les met en forme en compte-rendu structuré. Tout fonctionne 100 % en local sur la machine : aucune donnée audio ou texte ne quitte l'ordinateur, ce qui respecte le secret médical et le RGPD.

Conçu pour tourner en CPU pur, l'outil est portable sur des laptops modestes (16 Go de RAM suffisent).

②Comment ça marche

1
Transcription — Whisper medium (standard) convertit l'audio en texte, sur CPU.
  • Couche 1 · prompt d'orientation psy/psychiatrie (oriente le vocabulaire)
  • Couche 2 · dictionnaire de corrections déterministes (coquilles connues)
2
Structuration — Qwen2.5 3B (via Ollama) met le texte en sections cliniques.
  • Couche 3 · correction du jargon mal transcrit, à partir du contexte

Pourquoi un Whisper « standard » ? Les modèles fine-tunés médicaux testés tronquaient les dictées de plus de ~40 s et faisaient plus de fautes. Le Whisper standard gère le format long nativement ; la précision sur le jargon est rétablie par les trois couches de correction.

④Prérequis

  • Python 3.13 (ou 3.12)
  • Ollama installé (pour la structuration)

⑤Installation (une seule fois)

0. Récupérer le projet

git clone https://github.com/clemsous-cli/whispsy.git
cd whispsy

1. Environnement Python et dépendances

python -m venv .venv
.venv\Scripts\Activate.ps1

# torch en version CPU pure (plus légère)
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt

2. Convertir le modèle de transcription

Télécharge Whisper medium depuis Hugging Face (~3 Go la première fois) et le convertit en format optimisé CPU.

python convert_model.py

3. Récupérer le modèle de structuration

ollama pull qwen2.5:3b-instruct

⑥Utilisation

.venv\Scripts\Activate.ps1
python app.py

L'interface s'ouvre dans le navigateur. On y :

  • importe un fichier audio ou enregistre au micro ;
  • clique sur Transcrire → la transcription apparaît (modifiable : relisez-la) ;
  • (optionnel) clique sur Structurer en compte-rendu.

💡 Pas de micro sur un PC de bureau ? C'est normal (les tours n'en ont pas). Branchez un casque-micro, ou importez un fichier audio.

⑦Personnaliser

FichierÀ quoi ça sert
config.pyRéglages : modèle de structuration, langue, et surtout le prompt d'orientation à adapter à votre spécialité.
corrections.pyLe dictionnaire de corrections : ajoutez-y les erreurs récurrentes que vous repérez.
structure_report.pyLa consigne donnée au modèle de compte-rendu (sections, règles anti-invention).

Pour vérifier l'installation en ligne de commande : python test_pipeline.py votre_audio.wav (fournissez votre propre fichier audio ; aucun n'est inclus dans le dépôt).
